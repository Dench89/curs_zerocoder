Итеративный подход к разработке ↘️
Цель нашего мини-проекта — не писать всё сразу, а идти маленькими шагами через итерации:

Ядро — логика работы с заметками. Это backend, который отвечает за работу с файлами, самими заметками и функциями, доступными пользователю.
CLI — интерфейс через терминал, чтобы управлять ядром через команды в консоли.
Веб-интерфейс — возможность работать с заметками в браузере с полноценным UI.
Качество — работа с UX, устойчивостью программы и рефакторингом кода.

Пять шагов итеративной разработки ↘️
В результате урока формируется следующий шаблон работы с AI:

Формируем техническое задание — общий документ с описанием баз, структур, моделей файлов, логики работы и ожидаемого результата
Декомпозируем большую задачу на маленькие итерации
Генерируем код по каждой итерации
Проверяем результат — запускаем, тестируем
Улучшаем — вносим правки по UX/UI или логике и возвращаемся к шагу 4

Чем лучше техническое задание, тем стабильнее мы получаем результат.
Для написания ТЗ используем следующую структуру:

Контекст — описываем текущую систему, ограничения и цель, которую хотим достичь.

Реализация — указываем, что нужно сделать, какие технологии использовать.

Чем подробнее ТЗ, тем больше информации сможет понять нейросеть и тем точнее будет результат.

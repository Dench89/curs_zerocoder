# Итеративный подход к разработке

Цель — разрабатывать программу **не целиком**, а небольшими рабочими этапами (итерациями). После каждой итерации программа должна запускаться и работать.

---

# Этапы разработки

### 1. Ядро (Backend)
- Логика программы (как работает приложение).
- Работа с файлами (создание, чтение, изменение и удаление).
- Основные функции (возможности программы).

### 2. CLI (Command Line Interface)
- Управление через терминал (ввод команд в консоли).
- Проверка логики (тестирование функций без графического интерфейса).

### 3. Веб-интерфейс
- Работа через браузер (без использования терминала).
- Графический интерфейс (кнопки, меню, формы и окна).

### 4. Доработка
- UX/UI (удобство использования и внешний вид программы).
- Обработка ошибок (правильная реакция на неверные действия пользователя).
- Рефакторинг (улучшение структуры кода без изменения его работы).
- Оптимизация (ускорение работы и уменьшение потребления ресурсов).

---

# Цикл итеративной разработки

### 1. Составить техническое задание (ТЗ)
Создать документ с описанием проекта.

### 2. Разбить задачу на итерации
Разделить большую задачу на небольшие этапы.

### 3. Сгенерировать код
Попросить ИИ написать код только для текущей итерации.

### 4. Проверить результат
Запустить программу, протестировать и убедиться, что всё работает.

### 5. Улучшить
Исправить ошибки, доработать программу и перейти к следующей итерации.

---

# Шаблон технического задания (ТЗ)

## Контекст
Описать:
- текущую систему (что уже реализовано);
- ограничения (что нельзя использовать или изменять);
- цель (какой результат необходимо получить).

## Реализация
Указать:
- что необходимо сделать;
- какие технологии использовать;
- структуру проекта (файлы, папки, модели данных);
- ожидаемый результат (что должно работать после выполнения задачи).

---

# Главное правило

> **Одна небольшая задача → генерация кода → проверка → улучшение → следующая задача.**

Чем подробнее техническое задание, тем точнее и качественнее результат работы ИИ.
